Reponsive Web Design

With this exercise, I learned about Responsive Web Design from Tim Cole. The following are the three major steps:

Step 1: Set a meta viewport tag

< head >

< meta name"=viewport" content="width=device-width" content="initial-scale=1" >

< /head >

Step 2: Create a flexible layout / fluid grid Setup your site and your asset to re-size on their own according to the browser width.
  • Flexible Containers
  • Flexible Type
  • Flexible Margins and Padding
  • Flexible Images

Step 3: Use Media Queries in your CSS file(s) In terms of responsive design, using media queries is like assigning different stylesheets based on browser window size. It means you can style your website at various browser widths (such as for mobile or tablet), all in the same CSS file.

The trickest part to figure out was Step 3. While other wrappers and sections needed minor changes for certain effects to take place, the media queries didn't seem straight forward at first. I was able to find counter examples where the unordered list is laid out horizontally in a bigger display while they stack vertically when the display changes to a mobile screen. It took many trial and errors to see how the CSS attributes had its effects on the displayed material.
