Ruby: Enumerable Methods/ Mar 13, 2016

A way to search, sort, and more

From my understanding of iterations, enumerables are similar in that these set of methods allow an action to be performed on every object in the hash or array argument. More specifically, let's compare with a while loop of an array. For a while loop, you are iterating through each object within the array with a counter. As the counter matches the index with the length of the array, some logic is performed on each object. For an enumerable, you don't need a counter or a loop. The method called upon the array knows to inherently perform the logic specified to each object within the array.

Example Enumerable #map:

The .map method take an input of an enumerable object and a logic block. Let's take an array for our example.The enumerable object reprsents the place holder of each object within an array. The logic block can be anything that the user specify to be performed on every object within the array. The return output is an new array of values different from the original array. Note, .map is a non-destructive method. The original array still holds the same values. However, if you were to use the destructive method of .map!, then the original array will change.Code Embedding Source 

array = [1,2,3]

                array.map {|x| p x+1}
