Big O Complexity and Notation/ Apr 3, 2016

untangle the efficiency

The Big-O notation is used to describe algorithms. More specifically, it designates a function's response to inputs of different sizes and magnitudes at its worst case. In the example below, the loop goes through the entire array of three items before it finishes. If the logic inside the loop was true on the first item of the array, then the function takes less time to run. However, this is not the case, the function ran to the end of the array. In this example, the complexity is 0(n), and it read as "order of N". The example would be 0(3).

list = [1,2,3]
i=0
while i<list.length
 if i == 3
  return true
 end
 i+=1
end
