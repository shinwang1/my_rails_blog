What is a class in Ruby and how to create one

Within Ruby, objects have classes. If you were to look up classes from online material, you'll see that the analogy often use is a blueprint for creating objects. Within your IRB terminal, you can check on classes that are built into Ruby. For example, if you type 1.class or "word".class, you'll get a result of Fixnum and String respectively. Ruby knows the behaviors of number and strings because that information is contained in its respective classes.

Example of creating a class:

Consider the code below. The Class Cooking_Spaghetti performs a specific function. Once initialized, the goal of the Class is to tell the user if they have finished cooking their spaghetti. If you initialize the program without any cups of water, then you'll get an error. When you call the water_temperature method, you will be told whether it's done cooking. The instance variables are @water and @start-time. @start-time is passed into the boiled? method to check the delta between when the program was initialized and the current time. When the time is above 60 secounds, then you will be notified that you're done cooking. The stop_time variable is a local variable since it's only accessed within the boiled? method. Class Cooking_Spaghetti
    def initialize(water)
        if water.empty? == true
            raise ArgumentError.new("We need water to boil spaghetti")
        else
            @water = water
            @start_time = Time.now
        end
    end

    def water_temperature
        if boiled? == true
            p "Your #{@water} of water is boiled and your Spaghetti is ready."
        else
            p "We need to keep boiling."
        end
    end

    def boiled?
        stop_time = Time.now
        if stop_time - @start_time > 60
            return true
        else
            return false
        end
    end
 
end
##### Driver Code
cook = Cooking_Spaghetti.new(5)
cook.water_temperature # this method will tell you if you're done cooking.
